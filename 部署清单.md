# 公司内部 Chat 工具 · 零基础部署清单

> 适用：公司内部分发链接，每人自备 API Key  
> 预计耗时：第一次约 **1～1.5 小时**（含注册账号）

---

## 部署前准备

**模型列表已预填好**，无需改代码。同事首次打开需自行填写 **Base URL** 和 **API Key**（可向公司技术或中转站文档索取 Base URL）。

若以后要增删模型，在 `index.html` 里搜索 **`BUILD_DEFAULTS`** 修改 `models` 列表即可。

---

## 第一步：注册 GitHub（约 10 分钟）

GitHub = 放代码的网盘，Vercel 会从这儿自动部署。

1. 打开 https://github.com/signup
2. 填邮箱 → 设密码 → 起用户名 → 完成验证
3. 登录后看到首页即成功

---

## 第二步：把项目上传到 GitHub（约 20 分钟）

### 2.1 新建仓库

1. 登录 GitHub，右上角 **+** → **New repository**
2. **Repository name** 填：`company-chat`（名字随意，英文即可）
3. 选 **Private**（公司内部建议私有）
4. **不要**勾选 "Add a README file"
5. 点 **Create repository**

### 2.2 上传文件

1. 在新仓库页面点 **uploading an existing file**（或 **Add file** → **Upload files**）
2. 把本地 `chat-ai-all` 文件夹里的**所有文件**拖进去，包括：
   - `index.html`
   - `api` 文件夹（整文件夹拖入）
   - `cloudflare-worker.js`（可选，长回复用）
   - `serve.py`、`serve.rb`、`启动.command`（可选，本地用）
3. 下方 **Commit message** 可填：`首次上传`
4. 点 **Commit changes**

### 2.3 确认文件齐全

仓库里应能看到：

```
index.html
api/chat/completions.js
...
```

---

## 第三步：注册并部署 Vercel（约 20 分钟）

Vercel = 免费托管网页，自动生成 `https://xxx.vercel.app` 链接。

### 3.1 注册 Vercel

1. 打开 https://vercel.com/signup
2. 选 **Continue with GitHub**，授权登录

### 3.2 导入项目

1. 登录后点 **Add New…** → **Project**
2. 在列表里找到 `company-chat`（你刚建的仓库）→ **Import**
3. **Configure Project** 页面一般不用改，直接点 **Deploy**
4. 等待约 1～2 分钟，出现 **Congratulations** 即成功

### 3.3 复制链接

1. 点 **Continue to Dashboard**
2. 找到 **Domains** 一栏，类似：`https://company-chat-xxx.vercel.app`
3. **复制这个链接** — 这就是发给同事的地址

### 3.4 自己先验收

1. 用浏览器打开链接
2. 应自动弹出 **设置**
3. 填写 **Base URL** 和 **API Key**
4. **API 连接方式** 保持 **经 Vercel 本站代理**
5. 点 **测试 API 连接** → 显示成功 → 保存
6. 发一条「你好」试试

---

## 第四步：发给同事（复制即用）

把下面模板发到公司群 / 飞书 / 邮件：

---

**【公司内部 AI 聊天工具】**

链接：https://你的项目.vercel.app

**首次使用（约 2 分钟）：**
1. 打开链接
2. 在设置里填写 **Base URL** 和 **API Key**（模型列表已预填）
3. 连接方式选 **经 Vercel 本站代理**，点保存
4. 选模型，开始对话

**注意：**
- 请勿把 API Key 发给他人
- 对话存在本机浏览器，换电脑需重新填 Key
- 若回答中途断开，联系 @你的名字

---

## 第五步（可选）：长回复不断线

若同事反馈「回答写到一半就断了」（约 30 秒超时），再部署 Cloudflare Worker：

1. 注册 https://dash.cloudflare.com
2. 左侧 **Workers & Pages** → **Create** → **Create Worker**
3. 名称随意，点 **Deploy**
4. 点 **Edit code**，**删掉**编辑器里原有内容
5. 打开项目里的 `cloudflare-worker.js`，**全选复制**，粘贴到 Cloudflare 编辑器
6. 点 **Deploy**
7. 复制 Worker 地址，如 `https://chat-proxy.xxx.workers.dev`
8. 修改 `index.html` 里 `BUILD_DEFAULTS`：
   - `connectionMode` 改为 `'custom'`
   - `customProxyUrl` 填 Worker 地址
9. 重新上传到 GitHub（第二步 2.2 同样操作，或让同事帮你 push）→ Vercel 会自动更新

---

## 以后怎么更新？

1. 改好 `index.html`（或其他文件）
2. 到 GitHub 仓库 → **Add file** → **Upload files** → 上传覆盖同名文件 → **Commit**
3. 等 1～2 分钟，Vercel 自动部署
4. 群里说一句：**「工具已更新，请刷新页面（Mac：Cmd+Shift+R）」**

出问题可在 Vercel 控制台 **Deployments** 里点上一版的 **⋯** → **Promote to Production** 回滚。

---

## 常见问题

| 现象 | 处理 |
|------|------|
| Failed to fetch | 连接方式不要选「直连」，选 **经 Vercel 本站代理** |
| 测试连接失败 HTTP 401 | API Key 错误或过期 |
| 测试连接失败 HTTP 404 | 模型 ID 与中转站不一致，改 `BUILD_DEFAULTS` 里的 `modelId` |
| 回答到一半断了 | 做第五步，部署 Cloudflare Worker |
| 同事看不到新模型 | 让他们清除浏览器该站数据，或你改 `STORAGE_KEY` 后重新部署（会清空老用户本地记录） |
